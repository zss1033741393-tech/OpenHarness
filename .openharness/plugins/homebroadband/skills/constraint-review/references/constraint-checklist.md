# 约束校验评审清单

## 性能约束（违反 → blocker）
- [ ] 采集频率 × 指标数 × 用户数 ≤ 网关处理能力上限
- [ ] 聚合窗口 ≥ 采集间隔（否则数据不完整）
- [ ] 同时启用的诊断方法 ≤ 设备并发诊断上限
- [ ] 采集间隔不低于设备最小采集间隔（通常 ≥ 60s）
- [ ] 实时优化检查间隔 ≥ 采集间隔（避免数据不足就触发优化）

## 组网约束（违反 → blocker）
- [ ] 方案涉及的功能 ⊆ 设备型号支持的功能集
- [ ] 设备固件版本 ≥ 功能要求的最低版本
- [ ] 设备已纳管（配置下发通路可达）
- [ ] 闭环动作的前置条件与设备能力匹配（如 QoS 需设备支持流量标记）
- [ ] 漫游优化仅在多 AP 组网时启用

## 方案冲突（违反 → warning 或 blocker）
- [ ] 节能触发时段 ∩ 重点用户活跃时段 = ∅（冲突 → blocker）
- [ ] APPflow 策略 与 重点用户行为策略 无覆盖冲突（冲突 → warning）
- [ ] 高敏感业务期间 远程闭环自动触发 已禁用（违反 → blocker）
- [ ] WIFI漫游优化 与 覆盖优化 参数无矛盾（冲突 → warning）
- [ ] 多个闭环策略的执行窗口无时间重叠（重叠 → warning）
- [ ] 预测优化的提前量 ≤ 预测窗口（否则预测无意义 → warning）

## 方案完整性（违反 → info）
- [ ] 5 个方案维度均已生成（experience_perception / fault_diagnosis / remote_closure / dynamic_optimization / manual_fallback）
- [ ] 核心指标（CEI 阈值、响应 SLA）已设置且在合理范围
- [ ] 兜底方案的工单 SLA 与保障优先级一致
- [ ] 通知渠道已配置（至少一种通知方式）
