# 用户类型画像详情

## 直播用户
- **核心需求**: 上行带宽稳定、低丢包
- **典型行为**: 集中在晚间时段（19:00-24:00），上行流量大
- **关键指标**: 上行丢包率 < 0.1%，上行抖动 < 10ms
- **常见问题**: 推流卡顿、画面模糊、断流
- **设备特征**: 通常有专用直播设备，WIFI 5GHz 优先

## 游戏用户
- **核心需求**: 极低延迟、低丢包
- **典型行为**: 晚间和周末高峰，实时双向通信
- **关键指标**: RTT < 30ms，丢包率 < 0.05%
- **常见问题**: 延迟抖动、掉线、丢包导致操作延迟
- **设备特征**: PC/游戏主机，优先有线连接

## 办公用户
- **核心需求**: 视频会议质量稳定
- **典型行为**: 工作日白天为主（09:00-18:00）
- **关键指标**: MOS > 3.5，抖动 < 30ms
- **常见问题**: 视频卡顿、语音断续、屏幕共享延迟
- **设备特征**: 笔记本/台式机，多设备并发

## 教育用户
- **核心需求**: 视频流畅、不断线
- **典型行为**: 工作日白天 + 晚间辅导时段
- **关键指标**: 可用性 > 99.5%，下行带宽稳定
- **常见问题**: 视频缓冲、音画不同步、掉线重连
- **设备特征**: 平板/笔记本，WIFI 使用为主

## 普通家庭用户
- **核心需求**: 基本上网体验
- **典型行为**: 全天候，晚间流量高峰
- **关键指标**: 下行带宽达标、基本可用
- **常见问题**: 网速慢、WIFI 覆盖差
- **设备特征**: 多设备（手机/平板/电视/IoT）

## SOHO用户
- **核心需求**: 企业级网络可靠性
- **典型行为**: 工作日全天，需要VPN稳定
- **关键指标**: 可用性 > 99%，VPN 连接稳定
- **常见问题**: VPN断连、云桌面卡顿
- **设备特征**: 多台办公设备 + 网络打印机等

## 网络行为模式

### 周期性行为识别
- 每日定时直播 → 固定时段保障
- 工作日视频会议 → 工作日时段保障
- 周末游戏高峰 → 周末时段保障
- 定时断电重启 → 告警抑制

### 流量特征识别
- 持续高上行 → 直播/上传场景
- 突发双向流量 → 视频会议/游戏
- 持续高下行 → 视频观看/下载
- 小包高频双向 → 游戏/IoT
